# 数据流图说明（Data Flow Diagram）

数据流图（DFD）用于展示系统中数据的流动过程，以及外部实体、处理过程与数据存储之间的关系。

---

## 一、DFD 图用途

- 直观描述用户、系统和数据库之间的数据交互过程
- 帮助开发者理解各模块间数据输入输出的方向和内容
- 支持系统架构的分层设计与功能分析

---

## 二、数据流图层级划分

### 1️⃣ 顶层图（Level 0）

展示系统作为一个整体与外部交互实体之间的关系。

#### 主要元素：

- 用户（User）
- 商家（Seller）
- 管理员（Admin）
- 系统（HotMeal）
- 外部系统（例如 AI 推荐服务）

#### 主要数据流：

- 登录请求、订单提交、菜品推荐请求、数据展示请求等

---

### 2️⃣ 一级图（Level 1）

对系统模块进一步展开，明确模块之间的数据交互。

#### 主要子模块：

- 用户模块：注册、登录、信息维护
- 点餐模块：浏览菜品、下单、推荐
- 订单模块：创建订单、取消订单、支付处理
- 商家模块：商品管理、订单处理
- 管理模块：用户管理、统计图表展示
- 推荐模块：规则推荐、协同过滤推荐
- 数据存储：数据库中表结构（users, orders, dishes 等）

---

## 三、图例推荐符号（建议使用 draw.io 或 ProcessOn 绘图）

| 元素类型 | 图形     | 含义                             |
| -------- | -------- | -------------------------------- |
| 外部实体 | 矩形     | 系统外的参与者（用户、管理员等） |
| 处理过程 | 圆角矩形 | 系统中处理流程（如“下单处理”）   |
| 数据存储 | 双横线   | 数据库或文件存储                 |
| 数据流   | 箭头     | 数据在实体和处理过程之间的流动   |
